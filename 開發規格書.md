# 開發規格書 (Development Specifications)

## 1. 執行目標 (Agent Objective)
請身為一位資深 Python 後端工程師，閱讀本規格書與 `architecture.md` 後，依序建立所有模組碼與設定檔，並確保程式碼具備高模組化與容錯能力 (Error Handling)。

## 2. 模組開發詳細規格

### 模組 A：環境變數與安全設定
必須使用 `os.getenv()` 讀取以下變數。請建立 `.env.example` 提示使用者填寫：
*   `REDDIT_CLIENT_ID`
*   `REDDIT_CLIENT_SECRET`
*   `REDDIT_USER_AGENT`
*   `GEMINI_API_KEY`
*   `NOTIFY_TOKEN` (Line Notify Token 或 Telegram Bot Token)

### 模組 B：Reddit 抓取邏輯 (`reddit_scraper.py`)
*   **目標看板字典 (Target Subreddits Map)：**
    *   `AI 應用`: ["LocalLLaMA", "ChatGPTPro"]
    *   `傳統投資`: ["SecurityAnalysis", "algotrading"]
    *   `加密貨幣`: ["CryptoCurrency", "solana"]
    *   `國際政經`: ["geopolitics", "worldnews"]
*   **抓取條件：** 每個看板抓取 `time_filter="day"` 的 `top` 前 3 篇文章。
*   **錯誤處理：** 若某個看板抓取失敗 (例如 Reddit API 斷線或看板設為私密)，應紀錄 Error Log 並 `continue` 跳過該看板，不可讓整個程式崩潰。

### 模組 C：AI 處理邏輯 (`ai_processor.py`)
*   **模型選擇：** `gemini-1.5-flash` (具備速度快、成本低優勢)。
*   **System Prompt 設計 (極度重要，請直接實作此邏輯)：**
    ```text
    你是一個專業的國際新聞編譯與金融分析師。請將以下英文 Reddit 貼文的標題與內文，翻譯並總結為流暢的「台灣繁體中文」。
    請嚴格遵守以下輸出格式：
    1. 第一行：[繁體中文翻譯標題]
    2. 接下來用 3 個條列式重點 (Bullet points)，總結這篇文章的核心價值、結論或市場情緒。
    如果原文很短無內文，請僅憑標題推測其重點。不要輸出任何多餘的問候語。
    ```
*   **容錯機制：** 呼叫 API 需設置 Timeout。若遇到 Google API 阻擋 (Safety block) 或連線超時，請回傳預設字串："⚠️ AI 摘要生成失敗，請直接點擊原文連結閱讀。"

### 模組 D：推播與排版邏輯 (`notifier.py`)
*   將所有模組處理好的資料，組合成易讀的純文字報表。格式要求如下：
    ```text
    📊 每日全球情報晨報 (YYYY-MM-DD)

    🤖 【AI 應用】
    🔼 [推文數] 繁中標題
    - 重點 1
    - 重點 2
    - 重點 3
    🔗 連結: [Reddit URL]
    
    (以此類推接續其他分類...)
    ```
*   **長度限制：** 若總字數超過推播 API 限制 (如 Line Notify 的 1000 字限制)，需實作分段發送邏輯 (Chunking)。

### 模組 E：自動化排程 (`.github/workflows/schedule.yml`)
*   **觸發條件：** `cron: '0 0 * * *'` (對應台灣時間 UTC+8 的早上 8:00)。並保留 `workflow_dispatch:` 允許手動觸發測試。
*   **執行環境：** `ubuntu-latest`。
*   **步驟：** Checkout code -> Setup Python 3.10 -> Install dependencies (`pip install -r requirements.txt`) -> 注入 Secrets 作為環境變數 -> Run `main.py`。
