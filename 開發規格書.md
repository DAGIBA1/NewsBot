# 開發規格書 (Development Specifications)

## 1. 執行目標 (Objective)
建立一個高模組化、具備容錯能力的每日 Reddit 資訊監控與 AI 翻譯推播機器人。

## 2. 模組開發詳細規格

### 模組 A：環境變數與安全設定
使用 `os.getenv()` 讀取以下變數，透過 `.env`（本地）或 GitHub Secrets（CI）管理：
*   `REDDIT_CLIENT_ID`
*   `REDDIT_CLIENT_SECRET`
*   `REDDIT_USER_AGENT`
*   `GEMINI_API_KEY`
*   `TELEGRAM_BOT_TOKEN`
*   `TELEGRAM_CHANNEL_AI` — AI 應用分類的 Channel ID
*   `TELEGRAM_CHANNEL_INVEST` — 傳統投資分類的 Channel ID
*   `TELEGRAM_CHANNEL_CRYPTO` — 加密貨幣分類的 Channel ID
*   `TELEGRAM_CHANNEL_GEO` — 國際政經分類的 Channel ID

### 模組 B：集中設定 (`config.py`)
將所有可配置參數集中管理，避免 hardcode：
*   Subreddit 分類對應表 (`SUBREDDIT_MAP`)
*   分類 Emoji 對應 (`CATEGORY_EMOJI`)
*   分類 → Channel 環境變數對應 (`CATEGORY_CHANNEL_ENV`)
*   每看板抓取篇數 (`POSTS_PER_SUBREDDIT = 3`)
*   Gemini 模型名稱 (`GEMINI_MODEL = "gemini-2.0-flash"`)
*   去重複檔路徑 (`SENT_POSTS_FILE`)
*   Telegram 訊息長度上限 (`TELEGRAM_MESSAGE_LIMIT = 4096`)

### 模組 C：Reddit 抓取邏輯 (`reddit_scraper.py`)
*   **目標看板字典：**
    *   `AI 應用`: ["LocalLLaMA", "ChatGPTPro"]
    *   `傳統投資`: ["SecurityAnalysis", "algotrading"]
    *   `加密貨幣`: ["CryptoCurrency", "solana"]
    *   `國際政經`: ["geopolitics", "worldnews"]
*   **抓取條件：** 每個看板抓取 `time_filter="day"` 的 `top` 前 3 篇文章。
*   **去重複：** 接受 `sent_ids` 參數，自動跳過已推播的文章，並多抓以補足被過濾的篇數。
*   **錯誤處理：** 若某個看板抓取失敗，紀錄 Error Log 並跳過，不影響其他看板。
*   **資料結構：** 回傳 `list[CategoryPosts]`，每個 `CategoryPosts` 含分類名稱、Emoji 與 `list[RedditPost]`。

### 模組 D：AI 處理邏輯 (`ai_processor.py`)
*   **SDK：** `google-genai`（新版 SDK）
*   **模型：** `gemini-2.0-flash`（速度快、免費額度充足）
*   **System Prompt：**
    ```text
    你是一個專業的國際新聞編譯與金融分析師。請將以下英文 Reddit 貼文的標題與內文，翻譯並總結為流暢的「台灣繁體中文」。
    請嚴格遵守以下輸出格式：
    1. 第一行：[繁體中文翻譯標題]
    2. 接下來用 3 個條列式重點 (Bullet points)，總結這篇文章的核心價值、結論或市場情緒。
    如果原文很短無內文，請僅憑標題推測其重點。不要輸出任何多餘的問候語。
    ```
*   **內文截斷：** 過長內文截斷至 3000 字元以節省 token。
*   **容錯機制：** Timeout 30 秒。遇到 Safety block 或連線超時，回傳預設字串：「⚠️ AI 摘要生成失敗，請直接點擊原文連結閱讀。」

### 模組 E：推播與排版邏輯 (`notifier.py`)
*   **架構：** 1 個 Telegram Bot + 4 個 Channel，每個分類的晨報發送至對應 Channel。
*   **訊息格式：**
    ```text
    🤖 【AI 應用】每日情報 (2026-03-20)

    🔼 [500] 繁中標題
    [AI 摘要內容]
    🔗 https://reddit.com/r/...

    🔼 [300] 繁中標題
    [AI 摘要內容]
    🔗 https://reddit.com/r/...
    ```
*   **分段發送：** 單一 Channel 訊息超過 4096 字元時，按換行自動拆分為多段。
*   **重試機制：** 遇到 Telegram 429 (rate limit) 時，讀取 `retry_after` 等待後重試，最多重試 2 次。

### 模組 F：主程式整合 (`main.py`)
*   **執行流程：** 載入去重複記錄 → 抓取 Reddit → AI 摘要 → Telegram 推播 → 更新去重複記錄
*   **Logging：** 使用 Python `logging` 模組，記錄每個步驟的執行狀態。

### 模組 G：去重複機制
*   **儲存格式：** `data/sent_posts.json`，內容為已推播 post ID 的 JSON 陣列。
*   **持久化：** GitHub Actions 執行完畢後自動 commit 回存 repo（帶 `[skip ci]` 避免觸發循環）。

### 模組 H：自動化排程 (`.github/workflows/schedule.yml`)
*   **觸發條件：** `cron: '0 0 * * *'`（台灣時間 08:00）+ `workflow_dispatch` 手動觸發。
*   **執行環境：** `ubuntu-latest`。
*   **步驟：**
    1. Checkout code
    2. Install uv (`astral-sh/setup-uv@v5`)
    3. Setup Python (`uv python install`)
    4. Install dependencies (`uv sync`)
    5. Run tests (`uv run pytest tests/ -v`)
    6. 注入 GitHub Secrets 作為環境變數，執行 `uv run python main.py`
    7. Commit `data/sent_posts.json` 回存 repo

## 3. 測試規格

### Unit Tests (20 個測試案例)
| 測試檔案 | 測試對象 | 測試案例 |
|----------|---------|---------|
| `test_scraper.py` | `reddit_scraper.py` | 正常抓取、看板失敗容錯、空看板、去重複過濾、全分類遍歷 |
| `test_processor.py` | `ai_processor.py` | 正常摘要、API Timeout、Safety Block、空內文 |
| `test_notifier.py` | `notifier.py` | 正常發送、格式驗證（Emoji/標題/連結）、空貼文訊息、短訊息不分段、長訊息分段、rate limit 重試 |
| `test_config.py` | `config.py` | 分類 Emoji 對應完整、Channel 環境變數對應完整、看板非空 |
| `test_integration.py` | `main.py` | 全 mock 端到端流程驗證 |

### 執行方式
```bash
uv run pytest tests/ -v
```
